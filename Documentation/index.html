Router Documentation

This document explains how to define and handle various HTTP methods for routing in your PHP application. It also provides examples of dynamic parameters and proper route structure for better organization and maintainability.

Overview

The $router object allows you to define routes for your web application. Routes map HTTP methods (GET, POST, PUT, DELETE, PATCH) to specific controllers and actions.

Route Methods

The following methods are used to handle different types of HTTP requests. Each method corresponds to a specific action on the web server.

GET Route

Use $router->get to handle GET requests, which are used to retrieve data or view pages. 

$router->get('/post/{id}', 'Post::view');

POST Route

Use $router->post for handling POST requests, which are typically used for submitting data.

$router->post('/contact', 'Contact::submit');

PUT Route

Use $router->put for PUT requests, which replace the entire resource with the new data. 

$router->put('/post/{id}', 'Post::update');

DELETE Route

Use $router->delete to handle DELETE requests, which remove a specific resource.

$router->delete('/post/{id}', 'Post::delete');

PATCH Route

Use $router->patch for PATCH requests, which are used for partial updates to a resource. 

$router->patch('/post/{id}/title', 'Post::updateTitle');

Dynamic Parameters

Dynamic parameters allow you to capture values from the URL and pass them to the controller method.

$router->get('/post/{id}/{slug}', 'Post::view');

In this example, {id} and {slug} are dynamic parameters that will be passed to the Post::view method.

Example Routes

Here are some example routes that demonstrate the common HTTP methods in action:


$router->get('/', 'Home::index');          // Home page
$router->get('/about', 'Home::about');      // About page
$router->get('/send', 'EmailTest::sendContactEmail'); // Send contact email
$router->post('/contact', 'Contact::submit'); // Submit contact form

Conclusion

By defining clear and logical routes, your application becomes easier to maintain and expand. This documentation outlines the basic routes structure and provides an overview of dynamic parameters to help you better organize your routes.
