Routes Documentation

Overview

This documentation covers the various types of HTTP routes supported by the application, including GET, POST, PUT, DELETE, and PATCH routes. These routes are defined using the Router class and can be mapped to specific controller methods for handling HTTP requests.

GET Routes

GET routes are typically used to retrieve data or display pages. Below is an example of how a GET route is defined:

$router->get('/route', 'Controller::method');

Example: Displaying a list of posts:

$router->get('/posts', 'Post::index');

POST Routes

POST routes are used to submit data to the server, such as form submissions or creating new resources. Example:

$router->post('/contact', 'Contact::submit');

Example: Submitting a contact form:

$router->post('/contact', 'Contact::submit');

PUT Routes

PUT routes are used to update existing data on the server. Example:

$router->put('/post/{id}', 'Post::update');

Example: Updating a specific post by ID:

$router->put('/post/{id}', 'Post::update');

DELETE Routes

DELETE routes are used to delete resources from the server. Example:

$router->delete('/post/{id}', 'Post::delete');

Example: Deleting a specific post by ID:

$router->delete('/post/{id}', 'Post::delete');

PATCH Routes

PATCH routes are used for partial updates to resources. Example:

$router->patch('/post/{id}/title', 'Post::updateTitle');

Example: Updating only the title of a post:

$router->patch('/post/{id}/title', 'Post::updateTitle');

Dynamic Routes

Routes can contain dynamic segments, such as IDs or slugs, that are captured and passed as parameters to the controller method. Example:

$router->get('/post/{id}/{slug}', 'Post::view');

In this case, both {id} and {slug} are dynamic parameters that will be passed to the controller's view method as arguments.
